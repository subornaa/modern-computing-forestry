Self-assessment quiz

1. In the formula =B2 / $C$38, what does the dollar sign do?
  • Marks the cell as currency
  • Locks the reference so it does not move when the formula is copied
  • Indicates an error
  • Nothing — Excel ignores it
$C$38 stays as $C$38 when copied. Without the dollar signs, the reference shifts relative to the formula's new position.
2. Which function returns the number of non-empty cells in a range, including text cells?
  • COUNT
  • COUNTA
  • COUNTIF
  • LEN
COUNTA counts every non-empty cell (text + numeric + dates). COUNT only counts numeric cells.
3. To check that a lookup key appears exactly once in a reference table, which formula is best?
  • =VLOOKUP(A2, codes!A:B, 2, FALSE)
  • =COUNTIF(codes!A:A, A2) — should return 1
  • =IF(A2 = codes!A2, "OK", "not OK")
  • =ISNUMBER(VLOOKUP(...))
If the result is 1, the lookup is safe. If it is 2 or more, your VLOOKUP would silently return the first match.
4. A cleaning log should record:
  • Only the final cleaned data
  • Every change you made between raw and clean, with a reason
  • Only changes you cannot reverse
  • The data dictionary
A cleaning log captures every change with a reason. This is what lets you defend your numbers months later.
5. Which function is best for recoding the value in a cell to one of several exact-match labels (e.g., "P" → "Pre-sapling", "S" → "Sapling")?
  • VLOOKUP
  • A long chain of nested IF
  • IFS (or SWITCH)
  • SUMIF
IFS is designed for exact-match recoding. Cleaner and more readable than a long chain of nested IF.
6. You apply AutoFilter to hide rows where SumOfTotalHa < 1000. The other rows are not deleted — they are just hidden. True or false?
  • True — AutoFilter hides; it does not delete
  • False — AutoFilter deletes the hidden rows
AutoFilter is reversible. Sorting, by contrast, rearranges the rows permanently.
7. The formula =SUMIFS(H2:H180, F2:F180, "Jack Pine", D2:D180, "Mature") does what?
  • Sums every numeric value in column H
  • Sums the H values for rows where column F is "Jack Pine" AND column D is "Mature"
  • Returns "Jack Pine" if column D is "Mature"
  • Counts the rows that match both conditions
SUMIFS sums the first range, subject to one or more criterion ranges.
8. What is the main advantage of converting your data into a structured table (Ctrl+T) before writing formulas?
  • It makes Excel faster
  • Formulas use column names instead of cell ranges, and they auto-extend when rows are added
  • It encrypts the data
  • It is required for sorting
Structured tables make formulas self-documenting (=AVERAGE(tbl[col]) instead of =AVERAGE(H2:H180)) and self-extending.

  • The data is encrypted

  • PFTName alone

  • There is no key
The combination of region, ownership, seral stage, forest type, and age class uniquely identifies each row.
12. Press F4 while editing a cell reference. What does it do?
  • Closes the file
  • Cycles the reference through absolute / mixed / relative forms ($A$1 → A$1 → $A1 → A1)
  • Inserts a function
  • Triggers AutoFilter
F4 is the fastest way to add or remove dollar signs.
13. After cleaning the Ontario file, what should the file be named and where should it live?
  • Overwrite on_forest_statistics_1.xlsx in data/
  • Save as on_forest_statistics_1_clean.xlsx in outputs/
  • Email it to yourself

14. =AVERAGEIF(D2:D180, "Mature", H2:H180) returns —
  • The total area of mature forest
  • The mean SumOfTotalHa for rows where SERAL_NAME = "Mature"
  • The number of mature rows
  • The largest mature area
AVERAGEIF averages the third range, subject to the criterion.
15. Which is true about =VLOOKUP(A2, codes!A:B, 2, TRUE)?
  • It is the safest form of VLOOKUP
  • The TRUE means approximate match — dangerous if the lookup table is unsorted; can silently return the wrong value
  • It returns an error if the key is missing
  • It looks up the value in column B
Always use FALSE (exact match) for VLOOKUP unless you really mean approximate.
16. The mean of SumOfTotalHa in the Ontario file is much larger than the median. What does this tell you?
  • The data is wrong
  • The distribution is right-skewed — a few very large rows pull the mean up
  • The data is sorted
  • Half the rows are missing
When mean > median by a lot, the distribution is right-skewed. The median is the more honest measure of a "typical" row.
17. You convert your Ontario data to a structured table named ontario_data. Which formula references the SumOfTotalHa column correctly?
  • =AVERAGE(SumOfTotalHa)
  • =AVERAGE(ontario_data[SumOfTotalHa])
  • =AVERAGE("SumOfTotalHa")
  • =AVERAGE(@SumOfTotalHa)
Structured-table syntax uses table_name[column_name].
18. Which of these does NOT cause a VLOOKUP to fail?
  • A trailing space in the lookup key
  • The key is numeric but the lookup column is text
  • The lookup table is on the same sheet as the data
  • The exact-match flag is missing or TRUE
VLOOKUP works regardless of which sheet the lookup table is on. Trailing spaces, type mismatches, and the approximate-match default are the real culprits.
19. Why is sorting a structured table different from sorting a plain range?
  • Structured tables cannot be sorted
  • Structured tables keep formulas connected to the right columns even after rows are rearranged
  • Sorting a structured table makes Excel slower
  • There is no difference
Structured-table column references stay valid through sorts and filters because they refer to column names, not cell ranges.
20. Reflection. Describe one formula you wrote this chapter that did not work the first time. What was wrong, and how did you find out?
Common answers: VLOOKUP returned #N/A because of a trailing space in the lookup key · SUMIFS gave 0 because the criterion was a number formatted as text · a relative reference shifted incorrectly when copied down because I forgot the dollar signs on the denominator · IFS returned #N/A on some rows because the final TRUE catch-all was missing.

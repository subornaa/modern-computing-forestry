FRST 232 Transition Matrix

Week/Theme Legacy Approach Modernized Approach OER Data Strategy AI/Debugging Strategy
W1 - Course launch and Excel as a data-analysis workbench Orientation, software installation, Piazza setup, and the start of MS Word document processing. Computing was introduced through office-tool fluency before data analysis began. Begin with Excel immediately as a reproducible data-analysis workbench. Students import tabular files, inspect data types and number formats, define named ranges, create a data dictionary, and set up Posit Cloud for the later R transition. Word/reporting is no longer the opening technical focus. Use a small, authentic BC Ministry National Forest Inventory (NFI) extract in Excel. Students identify records, variables, units, missing-value conventions, species codes, and plot identifiers before doing analysis. Introduce the “Guided Answer” habit: inspect first, compute second, verify always. AI can explain an unfamiliar formula or code snippet, but students must verify against the spreadsheet structure and the data dictionary.
W2 - Excel cleaning, formulas, and safe referencing Word processing labs continued while Excel began with data types, number formats, text/CSV import, sorting, filtering, duplicate detection, data dictionaries, metadata, and spreadsheet navigation. Keep the Excel curriculum but modernize it around reproducibility: structured tables, explicit cleaning logs, safe relative/absolute cell references, summary statistics, conditional logic, and lookup tables. Use a larger NFI-style tree table with realistic issues: duplicate stems, missing DBH or height, inconsistent species labels, unusual plot IDs, and reference tables for species or site attributes. Students produce a one-page cleaning log. Guided hints focus on formula logic: “Check whether the reference should move when copied,” “Confirm the lookup key exists once,” and “Compare row counts before and after filtering.” AI is used to translate formula errors or explain lookup behavior, not to replace manual checks.
W3 - Excel visualization and PivotTables Excel visualization expanded through formula structure, random sampling, nested IF/AND/OR, lookup functions, bar/line/scatter charts, chart design, and real-world forestry/natural-resources examples. Excel becomes the bridge to later tidy data thinking. Students choose chart types based on questions, build PivotTables and PivotCharts, export figures with captions, and connect each chart to the underlying data dictionary and cleaning log. Students summarize NFI data by species, DBH class, site type, or management unit. PivotTables answer questions such as mean DBH by species, count of trees by size class, and basal-area summaries by plot group. Guided prompts target common analysis mistakes: wrong aggregation level, hidden filters, missing units, chart axes that imply the wrong comparison, and PivotTable fields in the wrong role. AI can critique chart clarity, but students must justify the final chart from the data question.
W4 - Excel wrap-up and bridge to reproducible programming Legacy Week 4 remained within Excel data visualization and Excel data-handling labs, including sorting, filtering, named ranges, formatted tables, exported results, and comparison of volume estimates. Use Week 4 to explicitly compare Excel strengths and limits: quick inspection versus reproducible audit trails, large-dataset limits, automation, and collaboration. Students install R, RStudio, and Quarto, create an RStudio Project, and learn a clean folder structure with the here package. Students export their cleaned NFI Excel table, data dictionary, and cleaning log as transition artifacts. The same dataset becomes the first R import, making the Excel-to-R handoff visible rather than abrupt. Guided Answer logic asks students to locate which step would be hard to audit in Excel and how a Quarto/R script would record it. AI is used for “what might break?” checks on file paths, data types, and hidden spreadsheet state.
W5 - Import and inspect forestry data with R and Quarto Legacy Week 5 focused on Excel PivotTables, PivotCharts, chart customization, summary functions, and midterm review. The midterm covered Word, Excel data handling, and Excel visualization. The modernized course moves the Excel midterm into the lab and starts R fundamentals in lecture: vectors, core data structures, reproducible CSV/Excel import, column-type inspection, missing-value checks, data dictionaries, and a first Quarto HTML report. Import the same curated NFI workbook/CSV used in Excel, then compare what R reads to what students saw in the spreadsheet. Students document column names, units, types, missingness, and expected keys in Quarto. Guided hints handle first R barriers: working directory, file path, package load, object name, column type, and rendering errors. Students use an AI prompt template that includes R version, package context, code tried, and the exact error, then verify with glimpse(), summary(), and row counts.
W6 - Clean and prepare forestry data in R Legacy Week 6 introduced RStudio and R Markdown, using R as a calculator, saving and knitting Rmd files, assigning variables, exploring object types, and midterm Q&A/review labs. Students now use tidyverse workflows to standardize names, filter, rename, arrange, handle missing values, detect outliers with domain knowledge, create derived variables, and maintain a reproducible cleaning log in Quarto. Use messy NFI inventory data with field-realistic problems: missing height, impossible DBH values, inconsistent species codes, plot-level metadata gaps, and derived variables such as basal area, age class, and size class. Guided Answer hints move from syntax to reasoning: “Check the column name after cleaning,” “Decide whether NA should be removed or flagged,” and “Confirm units before deriving basal area.” AI-generated fixes must be audited line by line and annotated in the cleaning log.
W7 - Reading Break Reading Break with no class. Reading Break with no class. Optional self-paced OER review can reinforce the Excel-to-R transition without adding new assessed content. Optional practice dataset: a tiny NFI sample for low-stakes import and inspection review. No new data concept is introduced. Optional Guided Answer self-checks revisit path errors, object names, missing values, and formula-to-code translation. AI use is reflective: students compare their own explanation to an AI explanation and identify any unsupported assumptions.
W8 - Summarize forestry data overall and by group Legacy Week 8 resumed R: import/export datasets, set/check working directory, explore environment, summarize datasets, use logical operators and subsets, create vectors/factors/matrices/data frames, do arithmetic, and introduce basic R plotting. The modernized course centers on tidy summaries: single-variable summaries, multiple-variable summaries, grouped summaries by category or location, interpretation of NA and NaN, and reporting group size with every mean. Use cleaned NFI data to calculate summaries by species, site type, forest management unit, or plot group. Students report counts, means, ranges, and missing-data diagnostics before interpreting ecological patterns. Guided hints catch common summary errors: forgotten na.rm, summarizing the wrong table, grouping left in place, small group sizes, and confusing missing values with zero. AI can suggest tidyverse verbs, but students must verify each summary with spot checks and n().
W9 - Reshape and combine forestry datasets safely Legacy Week 9 focused on base R/R Markdown graphical presentation: plot size, axis limits, margins, legends, annotations, plotting multiple datasets, subsets, side-by-side plots, and highlighting min/max/mean values. The new week teaches data shape and relational thinking before advanced graphics: long versus wide data, joins across tree records, plot metadata, site attributes, and diagnostic checks for duplicate keys and unexpected many-to-many relationships. Combine NFI tree-level measurements with plot metadata, species lookup tables, site attributes, management-unit information, and optional climate or disturbance summaries. Students track row counts before and after every join. Guided Answer logic emphasizes join safety: “Is the key unique in each table?”, “Did the number of rows change?”, “Which records failed to match?”, and “Are duplicates expected biologically or caused by data structure?” AI can explain join warnings, but students must produce pre/post diagnostics.
W10 - Visualize forestry data with ggplot2 Legacy Week 10 continued base R plotting and R Markdown export: saving plots, controlling resolution and dimensions, using legend(), text(), abline(), bar plots, percentage tables, rotated labels, and label-size control. Visualization shifts to the grammar of graphics: layered ggplot2, geoms chosen by question, facets for group comparison, custom themes, accessible color palettes, and publication-ready figure export. Students create NFI figures such as DBH-height scatterplots, DBH distributions, species composition bars, basal-area comparisons by group, and faceted plots across site type or management unit. Guided hints address aesthetic mapping, missing values, overplotting, scale choice, inappropriate geoms, and export dimensions. AI can propose alternative chart types or explain a ggplot2 error, but students must defend the final figure against the data question and accessibility criteria.
W11 - Spatial forestry data in R Legacy Week 11 still focused on R Markdown graphical presentation: pie charts, histograms, built-in datasets, boxplots, grouped boxplots, point and line graphs, subsetting seasonal trends, combining plots, and exporting visuals. The modernized course introduces spatial data with sf: points, lines, polygons, coordinate reference systems, reading BC spatial layers, inspecting geometry and attributes, joining attributes to features, filtering, summarizing, transforming, and reprojecting. Use BC NFI plot locations or aggregated NFI summaries with forest district, timber supply area, or management-unit boundaries. Students connect tabular inventory summaries to spatial features. Guided hints focus on spatial failure modes: missing CRS, wrong CRS, invalid geometry, latitude/longitude reversal, failed spatial joins, and maps that look plausible but are projected incorrectly. AI can help interpret CRS errors, while students verify with st_crs(), bounding boxes, and known BC geography.
W12 - Map forestry data and prepare project work Legacy Week 12 moved into presentation development and continued R Markdown graphical presentation labs, asking students to explore data patterns with graphs and summary statistics in structured R Markdown reports. Students make static thematic maps with ggplot2 and geom_sf(), layer multiple spatial datasets, add legends/labels/scale information, and build interactive leaflet maps when interactivity serves the question. Presentation topics are tied to lab-section groups and analysis outputs. Students map NFI-derived indicators such as mean DBH, basal area, species composition, or inventory completeness across spatial units. They create one static map and one interactive map that answer a forestry question. Guided Answer hints target layer order, missing geometries, color-scale misuse, legend clarity, accessibility, and whether interactivity improves interpretation. AI may debug layer-ordering or leaflet syntax, but students must verify map meaning and include alt text or descriptive captions.
W13 - Group presentations and portfolio outputs Legacy Weeks 12-14 were dominated by in-class group presentations on programming tools or code not covered in class, with peer review and reduced lab activity. Group presentations now showcase a forestry or natural-resources data-analysis question using the modern workflow. Deliverables include code, figures/maps, slides, and peer review. Public-domain data supports GitHub portfolio-style outputs. Groups use curated BC NFI data or compatible public natural-resources data to extend beyond lecture examples. Projects may combine NFI summaries with spatial boundaries, carbon-budget indicators, or management-unit comparisons. Guided review checklists ask whether the workflow renders from a clean project, whether joins and summaries are diagnosed, whether AI assistance is acknowledged, and whether another student can reproduce the output. AI is positioned as a code-review partner, not an author of unverified analysis.
W14 - Wrap-up and final exam preparation Legacy Week 14 finished presentations, observed holidays, and reviewed for a final exam focused on R/RStudio/R Markdown data handling and graphical presentation. The modernized wrap-up reviews the full reproducible pipeline: import, clean, summarize, reshape, join, visualize, map, render, and explain forestry data with R, tidyverse, Quarto, and spatial tools. Use a compact end-to-end NFI case study as final review: start from raw files, document cleaning, summarize by group, join metadata, produce one figure and one map, and render a Quarto report. Final Guided Answer ladder: read the error, identify the object/function/data step, inspect the data, reduce to a minimal example, test one fix, and document the verification. AI can be used during study to explain errors and generate practice variants, with explicit citation when it substantially contributes.
